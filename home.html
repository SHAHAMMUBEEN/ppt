Database Design

Multi-Database Architecture

The system uses 4 separate databases for efficient data management:

Database Purpose Key Tables
admin_system Admin users, success stories users, admin_users, success_stories
staff_system Staff management staff, staff_assignments
student_system Student information students
attendance_system Attendance tracking day_attendance, period_attendance, periods

Key Tables

  • users - User accounts
  • students - Student details
  • staff - Staff information
  • day_attendance - Daily attendance
  • period_attendance - Period-wise

Key Fields

  • Primary Keys: id fields
  • Foreign Keys: student_id, staff_id, admin_id
  • Unique: roll_no, rfid_uid
  • Indexes: For fast queries
